## 1. 目的と適用範囲

本書は、Virtual Infusion Pump Control Software(VIP-CTRL)の要求事項を IEC 62304 箇条 5.2 に基づいて定義し、後続の設計(SAD/SDD)・実装・試験の基礎とする。本版 0.1 は Inc.1(流量制御コア)範囲を対象とする。

### 1.1 適用範囲

- 対象ソフトウェア: VIP-CTRL(Inc.1 範囲)
- 対象バージョン: 0.1.0 以降
- 安全クラス: C
- 対象ユーザ: 医療従事者(医師・看護師・薬剤師)— ただし Inc.1 範囲では UI を含まないため、実質的に後続インクリメントの API クライアント(Inc.4 の UI 層)が本ソフトウェアの内部ユーザ

### 1.2 用語と略語(本書固有分)

| 用語 | 定義 |
|------|------|
| 流量(Flow rate) | 単位時間あたりの薬液送出量(mL/h が既定) |
| 投与量(Dose volume) | 総送出予定量(mL) |
| 投与時間(Duration) | 投与を完了するまでの予定時間(分または時間) |
| 定常注入(Constant infusion) | 設定流量を維持して薬液を連続送出する動作 |
| 仮想ポンプモデル | 実機ポンプ機構の挙動をソフトウェアで模擬するモジュール |
| 制御サイクル | 流量指令の更新周期(本プロジェクト: 100 ms を目標) |

既存の略語は CLAUDE.md §「英語略語の統一」表を参照。

## 3. システム要求事項からソフトウェア要求事項への展開(箇条 5.2.1)

| SYS ID | 要求事項 | 出典 |

## 4. ソフトウェア要求事項(箇条 5.2.2)

### 4.0 要求記載規約

- ID 形式: `SRS-NNN`(機能)、`SRS-PNN`(性能)、`SRS-I-NNN`(入力)、`SRS-O-NNN`(出力)、`SRS-IF-NNN`(インタフェース)、`SRS-ALM-NNN`(アラーム)、`SRS-SEC-NNN`(セキュリティ)、`SRS-UX-NNN`(ユーザビリティ)、`SRS-DATA-NNN`(データ)、`SRS-OPS-NNN`(運用・保守)、`SRS-REG-NNN`(規制)
- 優先度: **必須** = リリース必須、**推奨** = 望ましいが次リリース可
- 検証方法: **試験**(UT/IT/ST)、**レビュー**(設計/コードレビュー)、**解析**(静的解析・モデル検査)、**デモ**(システム試験での動作確認)
- Inc. 列: 要求がどのインクリメントで実装されるかを示す
- 各要求は **試験可能** であり、曖昧表現(「適切に」「必要に応じて」等)を避ける

### 4.4 ソフトウェア駆動のアラーム・警告・操作者メッセージ(箇条 5.2.2 d)

Inc.1 範囲では最低限の内部エラー通知のみ定義する。本格的なアラーム(IEC 60601-1-8 準拠)は Inc.2 で追補する。
